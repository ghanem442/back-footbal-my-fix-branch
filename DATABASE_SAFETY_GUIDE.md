# 🛡️ دليل حماية قاعدة البيانات

## ⚠️ تحذير مهم جداً

قاعدة البيانات تحتوي على بيانات المستخدمين والحجوزات. **أي خطأ قد يؤدي لفقدان كل البيانات!**

---

## ✅ الأوامر الآمنة (استخدمها دائماً)

### تشغيل السيرفر
```bash
npm run start:dev
```
✅ آمن تماماً - لا يمس قاعدة البيانات

### تطبيق Migrations الجديدة
```bash
npm run migrate:deploy
# أو
npm run prisma:migrate:deploy
```
✅ آمن - يطبق migrations بدون مسح بيانات

### إضافة بيانات تجريبية
```bash
npm run seed
# أو
npm run db:seed
```
✅ آمن - يضيف بيانات بدون مسح الموجود

### توليد Prisma Client
```bash
npm run prisma:generate
```
✅ آمن - يولد الكود فقط

---

## ⚠️ الأوامر الخطرة (استخدمها بحذر شديد)

### إنشاء Migration جديد
```bash
npm run prisma:migrate
```
⚠️ خطر متوسط - يسألك قبل التنفيذ
- استخدمه فقط عند تغيير الـ schema
- اقرأ التحذيرات جيداً

### Push Schema مباشرة
```bash
npm run prisma:db:push
```
⚠️ خطر عالي - قد يمسح بيانات
- **لا تستخدمه في production أبداً**
- استخدمه فقط في development للتجربة السريعة

---

## 🚨 الأوامر المدمرة (لا تستخدمها أبداً!)

### Reset كامل للـ Database
```bash
npm run prisma:migrate:reset
```
🚨 **خطر شديد - يمسح كل البيانات!**
- يحذف قاعدة البيانات كاملة
- يعيد إنشائها من الصفر
- **كل بيانات المستخدمين ستُفقد!**

**متى تستخدمه؟**
- فقط في development
- فقط إذا كنت متأكد 100%
- فقط بعد عمل backup

### الأوامر المباشرة (بدون npm)
```bash
# ❌ لا تستخدم هذه الأوامر مباشرة
npx prisma migrate reset
npx prisma db push --accept-data-loss
```
🚨 **خطر شديد - تتجاوز التحذيرات!**

## 🎯 السيناريوهات الشائعة

### سيناريو 1: عايز أشغل السيرفر
```bash
npm run start:dev
```
✅ بسيط وآمن

### سيناريو 2: عملت تغيير في schema.prisma
```bash
# 1. اعمل migration جديد
npm run prisma:migrate

# 2. اكتب اسم للـ migration
# مثال: add_user_avatar_field

# 3. السيرفر هيعيد التشغيل تلقائياً
```
✅ الطريقة الصحيحة

### سيناريو 3: عايز أضيف بيانات تجريبية
```bash
npm run seed
```
✅ آمن ومباشر

### سيناريو 4: السيرفر بيقول "table not found"
```bash
# 1. تأكد إن Docker شغال
docker ps

# 2. طبق الـ migrations
npm run migrate:deploy

# 3. شغل seed
npm run seed

# 4. شغل السيرفر
npm run start:dev
```
✅ الحل الصحيح

### سيناريو 5: عايز أمسح كل حاجة وأبدأ من جديد
```bash
# ⚠️ فكر مرتين قبل هذا!

# 1. اعمل backup أولاً (إذا كانت بيانات مهمة)
# 2. شغل reset
npm run prisma:migrate:reset

# 3. هيسألك تأكيد - اكتب "y"
# 4. انتظر 10 ثواني (فرصة للإلغاء)
# 5. البيانات ستُمسح

# 6. أعد البيانات الأساسية
npm run seed
```
⚠️ استخدمه بحذر شديد
